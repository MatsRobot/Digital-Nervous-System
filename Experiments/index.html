🎯 Validation Strategy

The MatsRobot system features multiple concurrent peripherals (CAN-Bus, dual I2C) which introduce significant noise and debugging risks. Stage 1 bypasses this complexity to answer two questions:

  • ✅
    Compilation: Can we successfully build for the ESP32-S3 via PlatformIO?
  • ✅
    Execution: Does the code physically toggle GPIO 13 on the silicon?
⚠️ Prerequisite: A successful blink test is required before adding any other wires or nodes to the system.

🔌 Hardware Reference

For this stage, only the LED and its current-limiting resistor are populated. You can ignore the CAN transceiver and I2C devices seen in the full schematic for now.

Figure 1: Complete architecture. For Stage 1, only components related to GP13 (Blue Wire, LED) are verified.

Connection Detail (GPIO 13):

  • Pin Location: Labeled '1' or 'GP13' on the DevKitC-1 header.
  • Wiring: Originates from GP13 → Current-limiting resistor → Anode (Long Leg) of the Heartbeat LED.

🚀 Developer Workflow

Deployment is handled via VS Code and the PlatformIO Core CLI.

# 1. Build: Verify code compiles without errors
pio run

# 2. Upload: Flash the firmware to the ESP32-S3
pio run --target upload

Observation: Upon success, the LED on GPIO 13 will blink at a 2.5Hz frequency (200ms ON / 200ms OFF). If this fails, subsequent complex stages cannot be attempted.
